# 主题代理系统 (ThemeAgentSystem)

## 概述

按小说类型注入专业化的写作规则、禁忌、场景模板和可插拔技能。让 AI 在写玄幻时像玄幻作者，写悬疑时像悬疑作者。

## 代理结构

每个主题代理包含 6 个组件：

| 字段 | 说明 |
|------|------|
| id | 唯一标识 |
| genre | 类型标识（xuanhuan / urban / scifi / mystery / romance / ...） |
| language | 语言（zh / en） |
| persona | 写作人设描述 |
| rules | 写作规则列表 |
| world_rules | 世界观约束描述 |
| taboos | 禁忌列表 |
| beat_templates | 节拍模板列表（每个包含 name、keywords 触发词、template 场景模板文本） |
| skills | 关联的技能 ID 列表 |

## 内置类型详细设计

### 玄幻 (xuanhuan)

- 人设：精通东方玄幻世界观构建的资深作者，擅长宏大的力量体系设计和热血战斗描写
- 规则：修炼等级严格递进不可跳级、战斗力量对比合理、境界突破是叙事事件而非数值提升、宝物获取需对等代价、宗门描写有层次感
- 世界观：修炼体系是核心规则，社会结构/权力格局/资源分配围绕修炼展开，强者为尊但受宗门规矩和天道法则约束
- 禁忌：不出现现代科技词汇、不使用游戏化表述、不让主角无脑碾压、不出现力量体系矛盾
- 节拍模板：突破场景（铺垫→契机→过程→危机→成功）、宗门大比（赛制→初赛→主角登场→强敌→苦战→转折→结果）
- 技能：cultivation_system, battle_choreography

### 都市 (urban)

- 人设：擅长现代都市题材，善于刻画复杂人际关系和社会现实
- 规则：对话贴近现实符合角色背景、情节符合现实逻辑、社会关系和职场描写有真实感、经济状况前后一致、感情线有足够铺垫
- 世界观：现代都市背景，遵循现实世界规则，除非设定包含超自然元素
- 禁忌：避免超自然元素（除非设定允许）、不出现不切实际的财富描写、角色行为符合社会身份、避免过度理想化
- 节拍模板：职场冲突（矛盾铺垫→正面交锋→暗中博弈→意外变数→阶段性结果）
- 技能：emotion_pacing

### 科幻 (scifi)

- 人设：硬科幻作者，注重科技设定内部自洽性和对人类社会的深层思考
- 规则：科技设定内部自洽、新科技需解释原理、科技对社会影响有合理推演、太空场景考虑基本物理规律、AI/外星文明避免过度拟人化
- 世界观：科技驱动世界运转，社会结构/伦理/战争因科技改变，尊重基本物理定律
- 禁忌：不出现魔法思维、不忽视光速限制（除非有超光速设定）、科技不是万能剧情工具、避免科技恐惧刻板叙事
- 节拍模板：科技发现（异常现象→初步调查→假设→验证→真相→伦理思考）
- 技能：world_building_scifi

### 悬疑 (mystery)

- 人设：精通推理小说，擅长精密线索布局和出人意料的反转
- 规则：公平推理原则（关键线索在揭示前对读者可见）、红鲱鱼不超过 30%、嫌疑人有合理动机和不在场证明漏洞、推理逻辑严密、真相揭示让读者有"原来如此"感
- 世界观：逻辑和证据是破案唯一途径，即使有超自然元素推理过程也必须遵循逻辑
- 禁忌：不在最后引入无铺垫的关键证据、不靠超能力破案、凶手动机不牵强、避免叙述性诡计滥用
- 节拍模板：案件发现（日常打破→案件发现→现场描写→初步调查→嫌疑人浮现→第一个误导方向）
- 技能：deduction_logic

### 言情 (romance)

- 人设：擅长情感描写，善于刻画细腻心理活动和复杂感情关系
- 规则：感情发展有足够铺垫和互动、情感反应符合性格和经历、误会和冲突不过于刻意、配角感情线合理发展、亲密场景有情感基础
- 世界观：情感关系是核心驱动力，外部事件服务于角色间的情感发展和内心成长
- 禁忌：避免突兀感情转变、不为制造冲突而 OOC、避免霸道总裁式刻板描写、女性角色不沦为纯粹被拯救对象
- 节拍模板：感情升温（日常互动→意外亲密→内心挣扎→关键事件→关系推进）
- 技能：emotion_pacing

## 可插拔技能详细设计

### 技能结构

每个技能包含以下内容：

| 字段 | 说明 |
|------|------|
| id | 唯一标识 |
| name | 技能名称 |
| description | 技能描述 |
| content.rules | 技能专属规则列表 |
| content.templates | 场景模板列表 |
| content.vocabulary | 推荐用词/术语列表 |
| content.anti_patterns | 避免的模式列表 |
| applicable_genres | 适用的类型列表 |

### 内置技能

| 技能 ID | 名称 | 适用类型 |
|---------|------|---------|
| cultivation_system | 修炼体系 | 玄幻/仙侠/修仙 |
| battle_choreography | 战斗编排 | 玄幻/武侠/都市异能 |
| deduction_logic | 推理逻辑 | 悬疑/推理 |
| emotion_pacing | 情感节奏 | 言情/都市 |
| world_building_scifi | 科幻世界构建 | 科幻 |
| power_system | 数值体系 | LitRPG/升级流 |

### 代理注入流程

ThemeAgentInjector 在章节生成时执行以下步骤：

1. 加载小说关联的主题代理
2. 基础注入：将 persona、rules、world_rules、taboos 注入上下文
3. 技能注入：加载关联技能的 rules 和 anti_patterns 追加到规则栈
4. 节拍模板匹配：检查当前蓝图的 summary/positioning 是否匹配某个模板的关键词，匹配则激活该模板
5. 技能模板匹配：检查蓝图内容是否匹配技能的 vocabulary 关键词，匹配则追加技能模板

### 用户自定义

用户可以：
- 编辑内置代理的任何字段（修改后保存为用户版本，不影响内置版本）
- 创建全新的自定义类型代理
- 创建自定义技能并注册到任意代理
- 从模板市场（Phase 6）导入其他用户分享的代理和技能

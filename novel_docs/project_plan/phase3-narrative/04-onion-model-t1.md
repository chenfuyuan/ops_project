# 洋葱模型 T1 层

## 概述

Phase 1 实现了 T0/T2/T3 三层，跳过了 T1。Phase 3 补全 T1 层，实现完整的四层洋葱模型上下文管理。

## 完整四层模型

| 层级 | 内容 | Token 预算 | 裁剪策略 |
|------|------|-----------|---------|
| T0（永不裁剪） | 事实锁 + 角色锚点 + 活跃伏笔 | ~4K | 始终全量注入 |
| T1（可压缩） | 知识图谱子网 + 近期幕摘要 + 剧情线状态 + 章节意图 | ~6K | 超预算时 LLM 压缩或按相关度裁剪 |
| T2（动态水位线） | 最近 N 章全文 | ~12K | 从最远章开始裁剪，超出部分用摘要替代 |
| T3（优先牺牲） | 向量召回结果 | 剩余预算 | 按相关度排序，从低到高裁剪 |

注意：T2 从 Phase 1 的 ~18K 缩减到 ~12K，给 T1 让出空间。

## T1 层构建

MemoryEngine 的 _build_t1 方法构建 T1 上下文，包含四个部分：

1. 知识图谱相关子网：根据当前蓝图的 positioning 和 summary 查询相关三元组（限制 20 条），格式化为文本
2. 近期幕摘要：获取当前幕（act）内从起始章到上一章的摘要，逐章拼接
3. 剧情线状态：获取所有活跃剧情线，列出类型、名称和当前进度
4. 章节意图：如果提供了 ChapterIntent，列出 must_keep 和 must_avoid

## T1 超预算压缩策略

当 T1 内容超过 ~6K token 预算时，按优先级从低到高裁剪：

1. 裁剪知识图谱子网（按相关度排序，从低到高移除三元组）
2. 压缩幕摘要（保留最近的，移除较远的）
3. 剧情线和章节意图不裁剪（优先级最高）

## build_context 升级

MemoryEngine 的 build_context 方法升级为四层构建：

1. T0：永不裁剪，始终全量注入，计算 token 消耗
2. T1（Phase 3 新增）：构建后压缩到 min(6000, remaining * 0.25) 预算内
3. T2：动态水位线，预算为 remaining * 0.7
4. T3：优先牺牲，使用剩余预算

返回包含四层内容和各层 token 消耗的 ContextPayload。

# 章节蓝图增强

## 悬念单元可视化

章节按 3-5 章为一个"悬念单元"分组，前端可视化展示：

```
悬念单元 1 (Ch1-5): [铺垫组]
  ██░░░  ████░  █████  ██░░░  ████░
  Ch1:3  Ch2:4  Ch3:5  Ch4:2  Ch5:4

悬念单元 2 (Ch6-9): [高潮组]
  ████░  █████  █████  ███░░
  Ch6:4  Ch7:5  Ch8:5  Ch9:3
```

前端实现：
- 章节列表上方显示分组视图
- 每组用色块表示悬念密度
- 标注组类型：铺垫组（蓝色）/ 高潮组（红色）/ 缓冲组（绿色）
- 可拖拽调整分组

## 场景类型指导

蓝图新增 scene_types 字段，指导 Writer 在不同场景类型间切换风格。每个 SceneTypeGuidance 包含 type（场景类型）和 guidance（写作指导）。

### 四种场景类型

| 类型 | 写作指导 |
|------|---------|
| dialogue（对话） | 对话要有潜台词，体现角色间的权力关系和情感暗流。避免直白的信息传递，用动作和表情替代对话标签。 |
| action（动作） | 注重感官细节（视觉、听觉、触觉），节奏用短句加速。力量对比要合理，战斗结果要有代价。 |
| psychological（心理） | 内心独白展现认知冲突，用意识流或内心对话形式。避免直接告诉读者角色在想什么，通过行为暗示。 |
| environmental（环境） | 环境描写映射角色心理状态。不是单纯的风景描写，而是通过环境传递情绪和氛围。 |

### 注入方式

场景类型指导编入 Composer 的规则栈，Writer 在生成时参考。格式为"本章场景指导"段落，列出本章包含的场景类型及对应写作要点。

### 数据库变更

chapter_blueprints 表新增 scene_types 字段（JSON 类型），蓝图生成时 LLM 同时输出 scene_types。

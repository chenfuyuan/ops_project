# 实时通知

## Goal

在自动推进的重要节点上及时向用户反馈状态、风险和完成情况，避免自动化过程变成不可见黑盒。

## User Value

作者即使不盯着工作台，也能知道系统当前在做什么、是否需要自己介入。

## Success Criteria

- 自动化关键事件具备清晰的通知需求边界。
- 用户能够区分进度更新、风险提醒和完成反馈。
- 卡片不提前定义传输协议、消息格式或签名机制。

## Scope

- 自动化关键事件的反馈需求
- 实时状态与异步提醒的使用场景
- 与控制台和失败保护之间的联动边界

## Non-goals

- Webhook 协议细节
- SSE 或消息队列实现
- 通知模板设计

## Dependencies / Prerequisites

- [autopilot-execution](自动推进执行.md)

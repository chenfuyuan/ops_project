# 自动化失败保护

## Goal

为自动推进过程建立清晰的失败边界、暂停条件和恢复预期，避免系统在异常情况下持续扩大损失。

## User Value

作者可以放心开启自动执行，因为系统在风险升高时会优先保护作品和运行稳定性，而不是盲目继续。

## Success Criteria

- 自动化异常和风险处理被定义为独立需求。
- 用户可以理解哪些情况会触发暂停、熔断或人工介入。
- 卡片不提前落入熔断算法、重试策略或基础设施实现细节。

## Scope

- 自动推进过程中的失败保护需求
- 异常升级、暂停与恢复边界
- 与自动推进和通知能力的协作关系

## Non-goals

- 具体阈值定义
- 监控系统实现
- 运维平台集成

## Dependencies / Prerequisites

- [autopilot-execution](自动推进执行.md)

# Phase 5 - Automation

## Phase 目标

在质量和风格能力已经有稳定基线之后，让系统从人工主导逐步转向自动推进，并让用户能看到过程状态与风险。

## Phase 边界

- 只整理自动推进、风险保护、通知反馈和控制入口相关需求。
- 不提前写后台守护进程、事件总线、API 合约或数据库状态机细节。
- 自动化仍以单用户产品阶段为前提，不提前展开多租户或商业化设计。

## 进入标准

- 前置 phase 已经让创作质量具备可控基础。
- 团队需要减少人工逐步点击的负担，并能在自动执行时保留可控性。

## 退出标准

- 自动化所需的核心需求已拆成独立卡片。
- 自动推进、熔断保护、通知和控制台能力的关系清晰。
- 本阶段与 SaaS 平台化诉求保持边界分离。

## Phase 关系

- 前置：`consistency`, `narrative`, `style`
- 为后续解锁：`saas`

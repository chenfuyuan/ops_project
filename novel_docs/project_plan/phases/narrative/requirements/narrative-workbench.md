# 叙事工作台入口

## Goal

为张力分析、类型化指导、知识查询和上下文增强提供统一的使用入口，让叙事能力真正能被查看和操作。

## User Value

作者不需要分别寻找零散入口，就能在一个连续工作面中理解作品的叙事状态并做出调整。

## Success Criteria

- 本阶段核心叙事能力都具备可被查看或触发的入口需求。
- 工作台需求与底层叙事能力保持清晰依赖关系。
- 卡片不提前定义页面布局、组件树或 API 合约。

## Scope

- 叙事能力的统一查看入口
- 关键结果、提醒和人工操作的需求
- 与张力、类型化、知识图谱和上下文层之间的连接关系

## Non-goals

- 视觉设计系统
- 认证与权限设计
- 具体前后端协议

# 类型化写作指导

## Goal

让不同题材或主题的小说能够应用成体系的写作指导，减少所有作品都被同一套表达方式处理的情况。

## User Value

作者能够根据作品类型得到更贴合题材的创作辅助，而不是依赖通用型生成结果。

## Success Criteria

- 需求明确表达“不同题材需要不同创作指导”的边界。
- 后续可以围绕单一或多种题材能力独立进入 change。
- 卡片不提前规定技能系统、插件机制或 prompt 结构。

## Scope

- 类型化写作指导的产品需求
- 题材规则、禁忌和节拍约束的规划边界
- 与章节生成主流程之间的协作关系

## Non-goals

- 技能加载机制
- 模板语法
- 前端配置面板细节

## Dependencies / Prerequisites

- Phase 2 质量治理基线

# 知识图谱与事实关系查询

## Goal

把关键人物、事件和关系沉淀为可查询的知识层，帮助团队在长篇推进中更快定位事实关联。

## User Value

作者和系统都能更容易追踪“谁和谁发生了什么”“哪些关系已经成立”，减少信息散落导致的混乱。

## Success Criteria

- 知识关系沉淀被明确为独立需求，而不是附属于其他卡片的隐含细节。
- 用户可以把这项能力视为长期信息管理工具的一部分。
- 卡片不提前定义图谱 schema 或存储技术。

## Scope

- 关键事实关系的沉淀需求
- 围绕事实关系的查询和使用需求
- 与一致性治理和上下文管理的协作边界

## Non-goals

- 图数据库选型
- 抽取算法设计
- API 细节

## Dependencies / Prerequisites

- Phase 2 质量治理基线

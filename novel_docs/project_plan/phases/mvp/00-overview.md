# Phase 1 - MVP

## Phase 目标

建立人工主导的最小创作闭环，让团队能够从小说项目创建一路走到章节产出与章后状态沉淀。

## Phase 边界

- 只整理“先把端到端流程跑通”所需的基础需求边界。
- 本阶段卡片只表达产品与流程需求，不提前固化具体模块实现或技术方案。
- 不引入一致性治理、叙事增强、风格系统、自动驾驶和 SaaS 能力。

## 进入标准

- 团队确认项目仍以“先自用验证”作为当前方向。
- 当前最重要的问题是能否形成一个真实可走通的写作闭环。

## 退出标准

- MVP 所需的单需求卡片已经拆分清楚，并能分别进入未来的 change。
- 团队可以明确指出 Phase 2 才开始承担的一致性治理问题。
- Phase 1 的卡片不再依赖跨阶段的大设计文档才能被理解。

## Phase 关系

- 前置：无
- 为后续解锁：`consistency`

## 阅读路径

- 先看 [requirements.md](requirements.md) 了解本阶段需求索引。
- 再进入目标卡片，判断是否值得启动单个 change。

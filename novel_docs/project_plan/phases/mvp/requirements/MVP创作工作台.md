# MVP 创作工作台

## Goal

提供支持 MVP 闭环的基础前端工作台，让用户可以在一个连续界面中推进小说创建、规划和章节写作。

## User Value

作者不需要在多个临时工具之间切换，就能看到当前项目状态并完成主要创作操作。

## Success Criteria

- 前端需求覆盖 MVP 主流程中的关键页面与交互。
- 工作台与 API 入口、生成流程、状态查看之间的关系清晰。
- 卡片保持在使用体验层，不提前细化组件结构或视觉实现。

## Scope

- MVP 主流程需要的页面与操作入口
- 创作过程中的状态查看和人工确认需求
- 与后端能力的最小协同边界

## Non-goals

- 视觉设计规范
- 组件库选型
- 后续自动化或多用户控制台

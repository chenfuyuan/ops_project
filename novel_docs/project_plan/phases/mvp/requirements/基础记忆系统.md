# 基础记忆系统

## Goal

沉淀章节推进过程中持续需要被保留的核心状态，让后续写作不必每次从零重新组织上下文。

## User Value

系统能够记住前文的关键信息，作者也能更稳定地在长篇过程中延续角色、摘要和伏笔状态。

## Success Criteria

- 需求边界清晰表达“哪些信息需要被持续保留”。
- 后续章节生成和质量治理可以把这部分能力作为前置输入。
- 卡片没有提前滑向数据库设计或状态同步机制说明。

## Scope

- 章节摘要和关键创作状态的持续保留需求
- 角色、伏笔和创作上下文的基础追踪要求
- 与章节生成主流程之间的输入输出关系

## Non-goals

- 向量检索实现
- 事实锁治理
- 完整上下文分层方案

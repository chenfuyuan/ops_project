# 协作能力

## Goal

让多位参与者能够围绕同一部小说协同工作，同时保持清晰的访问边界和协作方式。

## User Value

创作不再局限于单个作者，编辑、协作者或审核者可以围绕同一项目参与工作。

## Success Criteria

- 协作被定义为独立需求，而不是账户功能的附带说明。
- 用户可以理解不同参与者如何进入并参与同一项目。
- 卡片不提前定义实时协同方案、权限矩阵细节或评论数据结构。

## Scope

- 多人参与小说创作的基础协作需求
- 共享、协同编辑和反馈入口的边界
- 与账户能力之间的依赖关系

## Non-goals

- 实时协同算法
- 评论系统实现
- 通知体系细节

## Dependencies / Prerequisites

- [account-access](账户与访问边界.md)

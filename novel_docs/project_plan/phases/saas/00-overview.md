# Phase 6 - SaaS

## Phase 目标

把已经验证过的单用户创作系统逐步演进为可服务多用户的产品形态，覆盖平台基础设施、账户体系、配额、协作与模板分发。

## Phase 边界

- 只整理平台化和商业化前置需求边界。
- 不在本阶段提前写数据库分区、认证实现、计费流程或部署架构细节。
- 需求卡片只说明为什么做、要达到什么结果以及与其他能力的依赖关系。

## 进入标准

- 自动化阶段的方向已清晰，团队开始评估从单用户工具向服务化产品演进。
- 已经具备值得被多人使用和运营的核心创作能力。

## 退出标准

- SaaS 所需的主要能力已经拆成独立卡片。
- 平台基础、账户、配额、协作和模板市场之间的关系清楚可导航。
- 本阶段需求没有回流成单用户阶段的设计补丁集合。

## Phase 关系

- 前置：`automation`
- 后续方向：平台化深化与商业化运营

## 阅读路径

- 先看 [requirements.md](requirements.md)。
- 再根据当前平台化优先级选择对应卡片。

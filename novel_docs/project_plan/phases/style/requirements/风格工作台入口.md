# 风格工作台入口

## Goal

为风格基线、漂移检测和去 AI 化规则提供统一查看与调整入口，让风格控制成为可操作能力而不是后台隐性逻辑。

## User Value

作者可以直观看到当前作品的风格状态，并在必要时做出调整或确认，而不是依赖不可见的自动处理。

## Success Criteria

- 风格能力都具备清晰的使用入口需求。
- 工作台能力与底层风格卡片之间的依赖关系清晰。
- 卡片不提前定义前端信息架构或交互细节。

## Scope

- 风格相关结果与提醒的查看入口
- 人工确认、比较或调整的需求边界
- 与漂移检测、规则约束的联动需求

## Non-goals

- 视觉规范
- 组件设计
- 完整接口定义

- [anti-ai-rules](去AI化规则.md)

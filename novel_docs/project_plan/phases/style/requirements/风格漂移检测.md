# 风格漂移检测

## Goal

监测长篇写作过程中风格逐步偏离既定基线的情况，帮助团队及时发现并处理文风失真。

## User Value

作者能够知道作品什么时候开始“不像原来那部作品”，而不是等累计很多章节后再整体返工。

## Success Criteria

- 风格漂移被定义为可持续观察的独立需求。
- 检测结果能够为人工调整或后续流程提供依据。
- 卡片不提前定义检测指标或重写方案。

## Scope

- 风格偏移的识别需求
- 漂移结果的反馈和提示需求
- 与风格基线和工作台之间的衔接关系

## Non-goals

- 统计模型细节
- LLM 评估 prompt
- 自动重写流程

## Dependencies / Prerequisites

- [style-fingerprint](风格指纹.md)

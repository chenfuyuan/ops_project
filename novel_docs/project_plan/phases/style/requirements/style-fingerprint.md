# 风格指纹

## Goal

为目标作品建立可复用的风格基线，让系统知道“这部作品想呈现什么样的表达气质”。

## User Value

作者可以把喜好的文风沉淀成持续可参考的输入，而不是在每次生成时反复口头描述。

## Success Criteria

- 风格基线被明确为独立需求，而不是附属注释。
- 后续风格控制能力都能把这张卡片视为前置。
- 卡片不提前展开特征提取方式或具体模型策略。

## Scope

- 风格基线的来源与使用需求
- 作品长期风格参照的规划边界
- 与漂移检测、规则约束之间的依赖关系

## Non-goals

- 风格特征算法
- 参考文本处理流程
- UI 配置细节

## Dependencies / Prerequisites

- Phase 2 质量治理基线

# Phase 4 - Style

## Phase 目标

在基础质量稳定后，进一步把注意力放在文本风格控制上，减少 AI 痕迹并保持长篇创作中的风格一致性。

## Phase 边界

- 只围绕风格识别、偏移监测、规则约束和使用入口整理需求。
- 不把本阶段扩展成泛化的叙事质量、自动化编排或用户系统设计。
- 卡片只表达需要达成的风格能力与边界，不给出具体规则库实现方式。

## 进入标准

- 一致性能力已明确，团队开始把“是否像 AI 写的”视为核心质量问题。
- 需要在长篇过程中持续衡量和校正文风偏移。

## 退出标准

- 风格控制相关需求已拆成可单独推进的卡片。
- 风格指纹、漂移检测、规则约束和工作台入口的关系清晰。
- 本阶段内容没有混入更上层的自动化或 SaaS 诉求。

## Phase 关系

- 前置：`consistency`
- 为后续解锁：`automation`

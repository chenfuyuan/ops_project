# 事实锁与状态治理

## Goal

为长篇创作中的不可逆事实和状态变化建立稳定治理边界，减少后续章节对既定事实的破坏。

## User Value

一旦关键剧情事实已经成立，作者和系统都能更稳定地在这个边界内继续创作，而不会轻易自我推翻。

## Success Criteria

- 卡片明确表达哪些状态需要被保护和持续追踪。
- 一致性审计与后续写作都能把这部分治理结果作为可信前置。
- 需求仍停留在治理边界层，而非具体数据模型说明。

## Scope

- 不可逆事实的保护需求
- 状态变更的可信更新边界
- 与审计、伏笔和后续叙事能力的依赖关系

## Non-goals

- 存储结构细节
- 校验框架选型
- 知识图谱建模

# 伏笔追踪

## Goal

持续记录伏笔的植入、推进和回收情况，避免长篇创作中出现伏笔遗忘或过久停滞。

## User Value

作者可以更安心地铺设长期线索，系统也能提醒哪些伏笔需要继续推进或已经可以收束。

## Success Criteria

- 伏笔从提出到回收的关键状态变化已经被需求化表达。
- 用户能够知道哪些伏笔仍处于开放状态以及是否需要推进。
- 卡片不提前展开具体状态字段或追踪算法。

## Scope

- 伏笔生命周期的阶段性表达
- 停滞、推进和回收的提醒需求
- 与章节生成和一致性治理之间的协作边界

## Non-goals

- 伏笔文本抽取方案
- 具体生命周期 schema
- 张力评分设计

- [fact-lock-governance](fact-lock-governance.md)

# AI Docs

This directory contains curated documentation for AI assistants working in this repository.

## What belongs here
- Stable coding rules that should shape generated code.
- Condensed project guidance that is useful across many tasks.
- Example patterns that show preferred implementation style.

## What does not belong here
- OpenSpec artifacts under `openspec/`. Those are loaded by the OpenSpec workflow when relevant.
- Temporary task notes or one-off debugging context.
- Duplicated copies of source docs when a condensed AI-oriented version is enough.

## Suggested reading order for AI
1. `rules/architecture_rules.md`
2. `rules/naming_rules.md`
3. `rules/tech_stack_baseline.md`
4. `rules/review_checklist.md`
5. `examples/README.md` when examples exist

## Maintenance rule
When source rules change in `docs/`, update the corresponding condensed files here so AI guidance stays aligned with the repository.
